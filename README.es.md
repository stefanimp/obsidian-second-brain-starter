# Starter para Segundo Cerebro en Obsidian

Un sistema práctico para organizar Obsidian como segundo cerebro sin convertir la toma de notas en burocracia.

Este repositorio contiene una versión limpia y segura para publicar de un flujo real de toma de notas basado en:

- pocas carpetas usadas como bandejas de trabajo;
- notas conectadas en vez de jerarquías rígidas;
- MOCs como índices vivos;
- Properties de Obsidian para dar estructura;
- consultas tipo Dataview/Bases para revisar y recuperar información;
- plantillas que obligan a pensar claro sin frenar la escritura;
- una capa opcional de memoria de proyecto para trabajar mejor con IA.

## Idea Central

Un segundo cerebro no es un almacén. Es un sistema de trabajo para pensar.

El objetivo no es guardar todo. El objetivo es transformar lo que lees, ves, estudias y vives en notas que puedas recuperar, conectar y reutilizar.

## Estructura Recomendada

```txt
0-Inbox/
1-Templates/
2-Notes/
3-Media/
Home.md
```

Las carpetas funcionan como bandejas físicas:

- `0-Inbox`: captura rápida y material sin procesar.
- `1-Templates`: plantillas.
- `2-Notes`: conocimiento real, MOCs, proyectos, fuentes y reflexiones procesadas.
- `3-Media`: PDFs, imágenes, dibujos, audios y archivos externos.

La estructura del conocimiento vive en enlaces, propiedades, aliases y MOCs.

## Tipos de Notas

| Tipo | Uso |
| --- | --- |
| `note` | Concepto, explicación, clase, idea o conocimiento reutilizable. |
| `moc` | Mapa de Contenido: índice que organiza notas relacionadas. |
| `source` | Libro, vídeo, artículo, curso, podcast o fuente externa. |
| `idea` | Hipótesis, idea de proyecto o iniciativa temprana. |
| `project` | Proyecto con objetivo, alcance, entregables y próxima acción. |
| `journal` | Reflexión privada que puede generar notas reutilizables. |
| `dashboard` | Página de navegación o revisión con enlaces y consultas. |

## Principios

1. Escribe con tus propias palabras.
2. Usa carpetas como bandejas, no como arquitectura principal.
3. Usa enlaces como etiquetas vivas.
4. Crea notas específicas cuando una idea lo merezca.
5. Usa metadatos suficientes, no infinitos.
6. Revisa el sistema cada semana.
7. No publiques tu vault personal.

## Memoria de Proyecto para IA

Si usas ChatGPT, Claude, Cursor, Codex u otros asistentes con tus notas, la solución no es pegar más contenido del vault. La solución es preparar mejor el contexto útil.

El starter incluye ahora un módulo opcional para:

- context briefs de proyecto;
- registros de decisiones;
- registros de acciones;
- notas de intentos fallidos;
- metadatos mínimos para mejorar recuperación sin burocracia.

Úsalo solo en proyectos largos o cuando notes que tienes que reexplicar el mismo contexto a una IA.

## Inicio Rápido

1. Copia los archivos de `templates/` en tu carpeta de plantillas de Obsidian.
2. Crea las carpetas `0-Inbox`, `1-Templates`, `2-Notes` y `3-Media`.
3. Crea un MOC `Areas` y un dashboard `Home` usando los ejemplos de `examples/`.
4. Captura lo nuevo en `0-Inbox` salvo que ya tenga una ubicación clara.
5. En la revisión semanal, borra, conecta o convierte las notas de Inbox en conocimiento real.

## Advertencia de Privacidad

No publiques tu vault real sin revisarlo. Las notas personales suelen contener nombres, emociones, ubicaciones, enlaces privados y contexto que no debe ser público.

Usa este repositorio como plantilla pública, no como espejo de tu base de conocimiento privada.
