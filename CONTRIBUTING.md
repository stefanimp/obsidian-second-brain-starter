# Contributing

Contributions are welcome if they keep the project simple, privacy-safe, and useful for real Obsidian workflows.

## Principles

- Keep templates lightweight.
- Do not add personal vault content.
- Prefer clear examples over complex abstractions.
- Avoid plugin sprawl.
- Explain why a field, folder, or query exists.

## Good Contributions

- Better template wording.
- Clearer setup instructions.
- Useful Dataview examples.
- Privacy checklist improvements.
- Translations that preserve the original meaning.

## Avoid

- Personal notes or real vault exports.
- Screenshots with private data.
- Large plugin-dependent systems.
- Overly complex metadata models.
- Generic productivity advice without practical examples.

## Pull Request Checklist

- [ ] The change does not include private data.
- [ ] The change keeps the starter kit usable for beginners.
- [ ] The change does not require unnecessary plugins.
- [ ] Markdown examples have been reviewed for clarity.
