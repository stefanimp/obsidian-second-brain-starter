# Starter per Secondo Cervello in Obsidian

Un sistema pratico per organizzare Obsidian come secondo cervello senza trasformare le note in burocrazia.

Questo repository contiene una versione pulita e sicura da pubblicare di un flusso reale di note-taking basato su:

- poche cartelle usate come vaschette di lavoro;
- note collegate invece di gerarchie rigide;
- MOC come indici vivi;
- Properties di Obsidian per dare struttura;
- query in stile Dataview/Bases per revisione e recupero;
- template che aiutano a pensare con chiarezza senza rallentare la scrittura.

## Idea Centrale

Un secondo cervello non è un archivio. È un sistema operativo per pensare.

L'obiettivo non è salvare tutto. L'obiettivo è trasformare ciò che leggi, guardi, studi e vivi in note recuperabili, collegabili e riutilizzabili.

## Struttura Consigliata

```txt
0-Inbox/
1-Templates/
2-Notes/
3-Media/
Home.md
```

Le cartelle funzionano come vaschette fisiche:

- `0-Inbox`: cattura rapida e materiale non ancora processato.
- `1-Templates`: template.
- `2-Notes`: conoscenza reale, MOC, progetti, fonti e riflessioni processate.
- `3-Media`: PDF, immagini, disegni, audio e file esterni.

La struttura della conoscenza vive in link, proprietà, alias e MOC.

## Tipi di Note

| Tipo | Uso |
| --- | --- |
| `note` | Concetto, spiegazione, appunto, idea o conoscenza riutilizzabile. |
| `moc` | Map of Content: indice che organizza note correlate. |
| `source` | Libro, video, articolo, corso, podcast o fonte esterna. |
| `idea` | Ipotesi, idea di progetto o iniziativa iniziale. |
| `project` | Progetto con obiettivo, scope, deliverable e prossima azione. |
| `journal` | Riflessione privata che può generare note riutilizzabili. |
| `dashboard` | Pagina di navigazione o revisione con link e query. |

## Principi

1. Scrivi con parole tue.
2. Usa le cartelle come vaschette, non come architettura principale.
3. Usa i link come tag vivi.
4. Crea note specifiche quando un'idea merita uno spazio proprio.
5. Usa metadati utili, non infiniti.
6. Rivedi il sistema ogni settimana.
7. Non pubblicare il tuo vault personale.

## Avvio Rapido

1. Copia i file in `templates/` nella cartella dei template di Obsidian.
2. Crea le cartelle `0-Inbox`, `1-Templates`, `2-Notes` e `3-Media`.
3. Crea un MOC `Areas` e una dashboard `Home` usando gli esempi in `examples/`.
4. Cattura il nuovo materiale in `0-Inbox`, salvo che abbia già una posizione chiara.
5. Durante la revisione settimanale, elimina, collega o promuovi le note dell'Inbox in `2-Notes`.

## Avviso sulla Privacy

Non pubblicare il tuo vault reale senza revisione. Le note personali possono contenere nomi, emozioni, luoghi, link privati e contesto che non dovrebbe essere pubblico.

Usa questo repository come template pubblico, non come copia della tua base di conoscenza privata.
