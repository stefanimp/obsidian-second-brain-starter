# Privacy Checklist Before Publishing

Do not publish your real vault without a privacy review.

## Check Content

- [ ] Remove personal names.
- [ ] Remove private relationships and emotional notes.
- [ ] Remove addresses, universities, workplaces, schedules, and locations if they identify you too precisely.
- [ ] Remove private URLs, tokens, API keys, calendar links, and shared documents.
- [ ] Remove screenshots containing personal data.
- [ ] Remove notes copied from paid courses or copyrighted books.
- [ ] Remove medical, financial, legal, or family information.

## Check Metadata

- [ ] Review `aliases` for personal names.
- [ ] Review `topics` and `area` for private links.
- [ ] Review `next_action`, `deadline`, and `repo` fields.
- [ ] Review file names.
- [ ] Review attachment names.

## Check Templates

- [ ] Templates must not contain personal links.
- [ ] Templates must not contain names of partners, friends, family, employers, or universities unless intentionally public.
- [ ] Templates should use generic placeholders like `area-name`, `Projects`, `Learning`, or `Work`.

## Better Strategy

Publish a public starter kit, not your real vault.

A public repo should teach the method while protecting the private knowledge base that made the method useful.
